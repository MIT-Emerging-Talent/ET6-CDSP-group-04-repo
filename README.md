## 🎯 Project Overview

This repository captures the process of our data science project, from
identifying the initial problem to delivering the final presentation. Our
approach emphasizes collaborative problem-solving, careful analysis, and clear
 communication to enable actionable insights.

## **Milestone 1: Problem Identification** - Due: June 16

### **Learning Goals_1:**

- Learn divergent/convergent thinking
- Identify solvable, meaningful problems
- Formulate a research question

### **Checklist_1:**

- [ ] Write a personal-experience based problem statement
- [ ] Conduct domain research (save in `0_domain_research/`)
- [ ] Write domain summary (consider systems thinking)
- [ ] Formulate an actionable research question
- [ ] Maintain/update all planning docs from Milestone 0
- [ ] Update `README.md` with research question
- [ ] Write a retrospective for the milestone
- [ ] Create a labeled Git tag (before the deadline)
- [ ] Complete milestone survey

---

## **Milestone 2: Data Collection** - Due: June 30

### **Learning Goals_2:**

- Understand domain modeling with data
- Study data quality and availability
- Document data sources and processes

### **Checklist_2:**

- [ ] Collect and clean data
- [ ] Save all data processing scripts
- [ ] Create training/validation split (if needed)
- [ ] Host dataset publicly (repo or external link)
- [ ] Write a non-technical domain modeling explanation in `README.md`
- [ ] Document dataset source, structure, limitations, reproducibility
- [ ] Write a retrospective for the milestone
- [ ] Create a labeled Git tag (before the deadline)
- [ ] Complete milestone survey

---

## **Milestone 3: Data Analysis** - Due: July 21

### **Learning Goals_3:**

- Match analysis to data/question
- Address uncertainty and errors
- Learn to interpret and accept results

### **Checklist_3:**

- [ ] Perform data analysis (simple is OK!)
- [ ] Save all scripts for replicating analysis
- [ ] Write non-technical findings summary (include uncertainty & visuals)
- [ ] Write technical analysis explanation (method, flaws, alternatives)
- [ ] Write a retrospective for the milestone
- [ ] Create a labeled Git tag (before the deadline)
- [ ] Complete milestone survey

---

## **Milestone 4: Communicating Results** - Due: August 11

### **Learning Goals_4:**

- Translate insights into action
- Communicate clearly to a defined audience
- Handle uncertainty and limitations

### **Checklist_4:**

- [ ] Define target audience, capabilities, constraints
- [ ] Create a communication plan: what to teach, how to reach them, expected actions
- [ ] Build communication artifact (e.g., website, poster, pitch, campaign, etc.)
- [ ] Justify why this medium fits your audience
- [ ] Write a retrospective for the milestone
- [ ] Create a labeled Git tag (before the deadline)
- [ ] Complete milestone survey

---

## **Milestone 5: Final Presentation** - Due: August 25

### **Learning Goals_5:**

- Practice public speaking
- Reflect on team process and results
- Synthesize key learnings

### **Checklist_5:**

- [ ] Record a 1-minute pitch of your solution
- [ ] Prepare a 3-minute group presentation covering:
  - [ ] Project journey
  - [ ] Challenges faced
  - [ ] Lessons learned
  - [ ] Advice to future groups
- [ ] Add link to final presentation in `README.md`
- [ ] Write a retrospective for the milestone
- [ ] Create a labeled Git tag (before the deadline)
- [ ] Complete milestone survey

## Setup

1. Clone this repository:

   ```bash
   git clone https://github.com/MIT-Emerging-Talent/ET6-CDSP-group-04-repo.git
   cd ET6-CDSP-group-04-repo
   ```

2. Install dependencies:

   ```bash
   pip install -r requirements.txt
   ```

3. Set up your development environment following our [contribution guidelines](docs/CONTRIBUTING.md)
